# TalentFlow AI｜AI 面试人才库系统 MVP

本项目是为 AI 产品经理（Vibe Coding 方向）岗位定制的第一版 MVP，用于展示基于真实业务场景的需求拆解、产品流程设计、AI 工作流设计和快速原型落地能力。

TalentFlow AI 是一个面向 HR 的 AI 面试人才库系统，用于管理候选人面试流程、记录评价与结果，并通过 AI 辅助分析候选人优劣势和招聘数据。

## 项目定位

这是第一版可在线体验的 MVP 原型，不是完整生产系统。第一版以 HR / 招聘负责人视角为主，跑通候选人录入、面试安排、面试评价、结果跟进和数据分析闭环。

未来可扩展为 HR、业务面试官、部门负责人和管理者多角色协作系统。

## 已实现功能

- 候选人库：新增、编辑、删除、搜索、岗位筛选、状态筛选
- 简历录入：使用简历文本粘贴，不处理真实附件
- 面试安排：记录面试时间、面试官和候选人来源
- 面试评价：记录评分、评价、结果备注
- 状态流转：支持新候选人、已安排、待结论、面试通过、已淘汰、待报到、已报到、试用期、已归档
- 报到与试用期：记录报到时间和试用期情况
- Mock AI 分析：生成优势、不足、风险点和建议追问问题
- 数据看板：自动生成岗位面试情况、周度面试情况和月度面试情况
- 本地保存：使用 localStorage 保存演示数据
- 演示数据重置：一键恢复默认 Mock 数据

## MVP 边界

第一版暂不包含：

- 登录、权限和账号体系
- 真实数据库或后端服务
- 真实候选人隐私数据
- PDF / Word 简历上传解析
- 真实 AI API 调用
- 企业微信 / 飞书通知
- 面试官协作填写评价

AI 分析当前使用本地 Mock 规则实现，代码中预留了 `analyzeCandidate()` 函数，后续可替换为服务端真实 AI API。API Key 不应写入前端代码。

## 运行方式

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

构建生产版本：

```bash
npm run build
```

本地预览构建结果：

```bash
npm run preview
```

## GitHub Pages 部署

项目构建脚本已使用相对路径 `vite build --base=./`，构建后的 `dist` 目录可部署到 GitHub Pages。

基本流程：

```bash
npm run build
```

然后将 `dist` 目录发布到 GitHub Pages。也可以后续补充 GitHub Actions 自动部署工作流。
